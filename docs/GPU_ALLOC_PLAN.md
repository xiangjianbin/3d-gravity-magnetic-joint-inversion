# GPU 分配方案

## 硬件信息
- GPU: NVIDIA RTX 5000 Ada Generation
- 总显存: 33.9 GB (32 GB 标称 + 超额)
- 可用显存: ~31 GB (系统预留)
- CUDA: 12.1 (PyTorch 2.4.1+cu121)

## 模型显存测试结果

| Batch Size | 峰值显存(GB) | 占比% | 状态 |
|------------|-------------|-------|------|
| 2          | 0.20        | 1     | OK   |
| 32         | 0.23        | 1     | OK   |
| 64         | ~0.25       | 1     | OK   |
| 80         | 0.27        | 1     | OK   |
| 83         | 0.27        | 1     | OK (max) |
| 84         | —           | —     | OOM  |

## 分析

模型极其轻量（24.2M 参数），峰值显存仅 0.27 GB @ BS=83。
主要显存消耗在 U-Net 中间特征图（skip connections 复制）。

## 推荐配置

| 参数 | 推荐值 | 理由 |
|------|--------|------|
| 训练 batch_size | **16** | 安全值，留足余量给数据加载器、优化器状态、AMP scaler |
| 推理 batch_size | **32** | 推理时无梯度，可用更大 batch |
| 是否启用 AMP | **是** | 进一步节省显存（虽然当前不需要） |
| 是否使用梯度累积 | **否** | 当前 batch_size 已足够大；如需可设 accum=4 → effective_bs=64 |
| 有效 batch size | 16 | 对应论文 RTX 3070 8GB 的 batch_size=8 的 2x |

## 显存预算分解（估算 @ BS=16）

| 组件 | 预估显存 |
|------|---------|
| 模型参数 | ~0.1 GB |
| 输入张量 (16×2×40×40×20) | ~0.01 GB |
| 中间激活值 (U-Net skip) | ~0.15 GB |
| 梯度张量 | ~0.15 GB |
| 优化器状态 (Adam) | ~0.2 GB |
| 数据加载器预取 | ~0.5 GB |
| **总计** | **~1.1 GB** |
| **剩余** | **~30 GB** ✓ |

## 结论

RTX 5000 Ada 32GB 对此任务**严重过剩**。即使 batch_size=83 也只用 0.8% 显存。
建议使用 batch_size=16 配合 AMP 混合精度训练，确保训练稳定且快速。
