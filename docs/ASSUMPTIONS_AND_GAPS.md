# 假设与 Gap 清单

> 标记论文中未明确给出、需要推断或假设的参数

---

## Gap 1: 解码器通道数

- **问题**: 论文 Fig.2 只详细给出了编码器通道 (64→128→256→512→1024)，解码器只画了箭头
- **假设**: 解码器对称: 1024→512→256→128→64，每层先 Upsample 再 Conv
- **风险**: 低 — U-Net 对称解码器是标准做法
- **验证方式**: 冒烟测试检查输出形状

## Gap 2: LeakyReLU 的 gamma 参数

- **问题**: Eq.8 定义了 LeakyReLU 变体但未给 gamma 具体值
- **假设**: gamma = 0.01（标准默认值）
- **风险**: 低 — 对结果影响很小

## Gap 3: 学习率调度参数（Eq.12）

- **问题**: 公式形式给了但 β₀, β₁, β₂, lr₀ 均未指定数值
- **假设**:
  - lr₀ = 1e-3（Adam 常用起始值）
  - 使用 PyTorch CosineAnnealingLR 或 ReduceLROnPlateau 作为实际调度器
  - 不严格复现 Eq.12（该公式看起来是自定义变体）
- **风险**: 中 — 可能影响收敛速度和最终精度
- **缓解**: 监控训练曲线，如不收敛则调整

## Gap 4: Batch Size

- **问题**: 论文 Table II 未给出 batch size
- **假设**: 从显存倒推。RTX 5000 Ada 32GB → 预估 batch_size = 4~8
- **风险**: 低 — Phase 4 会实测确定最优值
- **验证方式**: GPU_ALLOC_PLAN.md 会给出实测数据

## Gap 5: ASPP 各分支 filter 数

- **问题**: Fig.4 未标注每分支的输出通道数
- **假设**: 每分支 256 filters（与 DeepLab v3+ 一致），全局池化分支也是 256
- **风险**: 低 — ASPP 是成熟模块，参数标准化

## Gap 6: 径向网格操作（Radial Grid Operation, Task 3）

- **问题**: 这是论文的创新组件之一，描述最模糊。仅提到"radial grid operations (left/right)"
- **假设**:
  - 先实现为标准的 3D 卷积 + 池化结构
  - "径向网格"可能指从中心向外的方向性池化或特殊卷积核
  - 初版先用 Conv3d + MaxPool3d 替代，效果不够再细化
- **风险**: 中高 — 这是核心创新点，实现偏差可能影响 Task 4/5 质量
- **缓解**: 先跑通 pipeline，再针对性优化此模块

## Gap 7: 地磁参数（合成数据）

- **问题**: 合成数据的磁法正演需要 I(倾角), D(偏角), F(强度)，论文未指定
- **假设**: I=45°, D=0°, F=50000 nT（中国地区典型中纬度值）
- **风险**: 低 — 只影响合成数据的绝对值，归一化后影响小
- **注**: 真实数据应用（Qianhe 铁矿）需要当地实际地磁参数

## Gap 8: 独立反演子网络内部细节（Fig.3）

- **问题**: Fig.3 显示了 4 层结构但每层具体卷积数不清晰
- **假设**: 每层 2 个 Conv3d + LeakyReLU，通道数递减: 256→128→64→32→1
- **风险**: 低 — 结构自由度大，只要输入输出形状正确即可

## Gap 9: 数据归一化方式

- **问题**: 论文说"normalized within [0,1]"但未说明具体归一化方法
- **假设**: Min-Max 归一化，分别对重力数据和磁数据进行:
  ```
  g_norm = (g - g_min) / (g_max - g_min)
  t_norm = (t - t_min) / (t_max - t_min)
  ```
- **风险**: 低 — 归一化方式对深度学习模型影响有限

## Gap 10: 观测网格到网络输入的下采样方式

- **问题**: 观测面 81×81 → 网络 40×40，如何下采样？
- **假设**: 双线性插值或简单取偶数行/列（81→40 约减半）
- **风险**: 低 — 下采样方式对最终结果影响较小
